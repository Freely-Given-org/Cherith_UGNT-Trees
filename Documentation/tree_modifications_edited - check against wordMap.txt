PROCEDURE

I looked for places where I discovered that the previous analysis was partially correct, or completely wrong.

For the number, it means:
1 = perfect match (or has same lemma)
2 = not a perfect match but is a fuzzy match
	a. text starts with over 70% same characters, or
	b. lemma starts with over 70% same characters, or
	c. same Strong's number, or
	d. same gloss

For my manual edits, I used 2 if it is not a perfect match, but I felt that it was advantageous to map them.
1. don't change the tree's skeletal structure (may change some nodes directly above the word)
2. has the same category
3. has the same Strong's number
4. has the same lemma

I decided to make the deletions and additions, rather than substitution (and thus mapped) if they don't satisfy the condition for 2

Just because it is a 2 mapping doesn't mean we should automatically use the NA28 gloss. We should check what kind of a mapping it is.
If it has a different lemma, Strong's number, then we probably want to find a gloss for the lemma or Strong's number, or use the existing gloss in the tree.
